* Restful 1) REST(Representational State Transfer): 데이터 전송 시 상태를 표현하는 방법 2) 브라우저에서 서버 쪽으로 요청할 때 데이터를 전송(queryString) 3) 데이터 전송 표현식 http://localhost:8000/main/add?username=hong&age=20&address=seoul => URL에서 add를 보고 실행하고자 하는 동작을 insert 하는 것으로 유추 가능 저장하는 데이터도 URL에서 username=hong&age=20&address=seoul 형식의 queryString 데이터로 사용할 수 있다. 4) Restful 방식 http://localhost:8000/main/username/hong/age/20/address/seoul => URL에 사용하는 데이터를 표현 => 실행하고자 하는 동작 URL에 표시 안하고 method="get/post/put/delete"로 표현 get: select post: insert put: update delete: delete
